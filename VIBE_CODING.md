# 🎵 VIBE CODING Скрипты

## Проблема
Claude Code потерял доступ к командам (git, cargo), что ломает "вайбкодинг" процесс.

## Решение
Скрипты в корне проекта для полноценного цикла разработки.

## 🛠️ Доступные скрипты

### `vibe.sh` ⭐ **Главный скрипт**
Полный цикл "вайбкодинга":
```
1. Компиляция
2. Проверка размера
3. Git коммит
4. Отправка на GitHub
```

**Использование:**
```bash
cd /Users/dmitriiromanov/claude_code/photomap
./vibe.sh
```

### `build.sh`
Быстрая компиляция с показом ошибок:
```bash
./build.sh
```

### `push.sh`
Отправка изменений на GitHub:
```bash
./push.sh
```

### `dev.sh`
Интерактивная разработка:
```bash
./dev.sh
```

## 🎯 Идеальный workflow

### В Claude Code:
1. **Я пишу код**
2. **Я объясняю изменения**
3. **Я готовлю к коммиту**

### В терминале:
```bash
./vibe.sh
```

Если есть ошибки → копирую в Claude Code → исправляю → `./vibe.sh` снова

## 📋 Сделать скрипты исполняемыми:

```bash
chmod +x /Users/dmitriiromanov/claude_code/photomap/*.sh
```

## 🚀 Преимущества

✅ **Восстанавливает "вайбкодинг"** - я правлю ошибки, вы запускаете
✅ **Минимальное копирование** - только при ошибках компиляции  
✅ **Полный цикл** - от кода до GitHub в одном скрипте
✅ **Размер под контролем** - автоматически показывает размер бинарника

## 💡 Будущее

Когда Claude Code вернёт доступ к командам - скрипты не понадобятся.
Но пока это **лучший способ сохранить продуктивность**.
